# Gobernanza del design system — Dimanche

> Cómo se usa y se mantiene el sistema sin que se desarme.

## Do's & Don'ts
**SÍ:** foto real siempre de base · 1 ícono real por pieza · titular corto con ingenio · Niveau Grotesk · paleta cobalto/crema · badge de cierre · scrim para legibilidad · mucho aire.

**NO:** producto inventado por IA · íconos dibujados a mano / fuera del set · naranja · emojis · más de 2 pesos tipográficos por pieza · titular largo o marketinero ("imperdible", "el mejor") · texto pegado al borde · texto sin contraste sobre foto cargada.

## Accesibilidad / legibilidad
- Contraste texto/fondo **≥ 4.5:1** (texto chico) · **≥ 3:1** (titular grande).
- Texto sobre foto → **scrim siempre**.
- Tamaño mínimo legible: el cuerpo nunca por debajo del Epígrafe (24 px @1080).

## Naming y orden de assets (Drive)
- Elementos gráficos → `00_Marca/Elementos_Graficos` (cobalto + crema; las naranja están deprecadas).
- Piezas finales → `03_Piezas/Redes` (o por canal).
- Nombre claro: `[formato]-[tema]-[fecha]-vN` (ej. `story-criollos-calientes-2026-06-v1`).
- El `Manual_de_Marca.pdf` viejo y los assets naranja = **histórico, no usar**.

## Versionado y mejora continua
- El design system se **versiona** (esta es V1). Cambios → actualizar el doc + **MAN-021** (Notion) + avisar.
- **Mejora continua:** lo que funciona en performance (Numa) actualiza estas reglas; revisión periódica. Las tendencias (radar) ajustan tácticas, **no** el norte.
