# Componentes — Dimanche

> Las **piezas reutilizables** que se combinan en cada formato. Usan los fundamentos (`foundations.md`) + los elementos reales del Drive. Se **componen por código** (Satori/PIL para estáticas, Remotion para video; ver `motor-de-composicion.md`) — **nunca en Canva**. Encima de esto van los **templates por formato** (capa siguiente).

## 1. Badge de logo
El **badge circular de la "D"** ("Panadería · Pastelería"), cobalto sobre blanco (o blanco sobre foto). Va **abajo-centro**, ~120–160 px (sobre 1080), con clear space. Es la **firma de cierre** de casi toda pieza. Archivo real: `00_Marca/Elementos_Graficos/Blue/blue d.*` (o la variante crema).

## 2. Lockup ícono + titular
La **unidad de arriba**: un ícono real (**silueta plana rellena en cobalto**, NO de línea; sobre la foto va en **blanco**, sobre fondo claro en cobalto; ~120 px) + el **titular** (Niveau, blanco, centrado) debajo, separados ~24 px. El ícono pega con el mensaje (sol = domingo, paraguas = lluvia, producto = …). Es el **gancho** de la pieza. (Estilo del set → `foundations.md`.)

## 3. Bloque de titular
**Titular** (Display/Titular) + opcional **antetítulo** (caps tracked) arriba y/o **subtítulo** debajo. Blanco sobre foto (con **scrim**) o cobalto sobre fondo claro. Centrado, ≤ 2 líneas. (Escala y reglas → `foundations.md`.)

## 4. Banner / "cuadro"
**Bloque cobalto** (rectángulo redondeado) con texto blanco — para CTA, dato o etiqueta sobre la foto. Padding interno en múltiplos de 8. Ej: *"Pedí por Uber"*, *"Recién hecho"*.

## 5. Tag / pill (precio o CTA corto)
**Pastilla chica** (cobalto con texto blanco, o crema) para precio o acción corta. Esquinas redondeadas. Ej: *"Pedí ya"*, *"Hoy"*.

## 6. Sticker de producto
Las **etiquetas de producto** (budín, pan…) que ya existen en `04_Packaging/Stickers` — mismo sistema (Niveau + paleta). Para packaging y mostrador.

## Cómo se combinan (la receta base)
La mayoría de las piezas =
**lockup (ícono + titular) arriba · foto real full-bleed · badge abajo** — + banner/tag si hace falta.
Esa es la plantilla base (`design-system.md`). Cada formato la adapta → **templates** (capa siguiente).
